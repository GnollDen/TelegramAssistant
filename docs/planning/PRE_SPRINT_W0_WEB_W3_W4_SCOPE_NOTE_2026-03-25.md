# Pre-Sprint W0 Web W3/W4 Scope Lock Note

## Date

2026-03-25

## Purpose

Зафиксировать scope "normal working web" для W3/W4 без сдвига в public-product polish.

## W3 Scope Lock (Clarification and Deep Review)

Входит в W3:
1. Расширенный clarification review flow.
2. Long-form user context/corrections в операторском процессе.
3. Evidence drill-down для case/artifact review.
4. Timeline/people/date context navigation вокруг кейса.
5. Linked cases/artifacts traversal.
6. History/reopen/outcome visibility where practical.

Не входит в W3:
- широкая visual-polish программа
- публичные продуктовые сценарии
- migration к другой backend architecture

## W4 Scope Lock (Usability and Hardening)

Входит в W4:
1. Usability cleanup:
- filters/sorting
- loading/empty/error states
- action confirmation UX
2. Web-specific verification tightening (smoke + manual baselines).
3. Basic rendering/performance stability improvements.
4. UX coherence для daily operator use.

Не входит в W4:
- маркетинговый/public UI polish
- product-surface expansion beyond internal operator needs
- broad feature wave вне W1-W4 web track goals

## Normal Working Web Definition (Locked)

Normal working web достигается после W3+W4, если:
1. Complex clarification cases удобно обрабатываются в web.
2. Оператор может сделать evidence-first deep review без ручных обходов.
3. Повседневный workflow стабилен по состояниям и навигации.
4. Web verification baseline покрывает ключевые ежедневные риски.
